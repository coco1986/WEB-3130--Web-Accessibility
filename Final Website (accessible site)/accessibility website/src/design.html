Site Design

Site Design Process

Creating an accessible website is essential to ensure that users of all abilities can perceive, understand, navigate, and interact with the content effectively. One crucial aspect of accessibility lies in choosing appropriate fonts, font sizes, colors, and visual elements. These choices significantly impact the user experience for people with various disabilities, including visual impairments and cognitive challenges. Another crucial aspect of accessibility involves the structure of the website because it directly affects how users, including those with disabilities, interact with and navigate through the content on a website.

Visual Design:

Visually I wanted this site to be simple. The color scheme was chosen by finding some colors that I liked by using the website Coolors and I chose colors that I thought would go well together and then using WebAIM: Contrast Checker, a free online application that allowed me to see color contrasts comparing a foreground color with a background color resulting in a pass ro fail contrast ratio as well as telling me if it passes normal text, large text, graphical objects and user interface components. I am pleased to say that the design of this site passes and meets all WCAG AA and WCAG AAA requirements in these categories. I made sure that each of the contrasts I chose had a contrast ratio of 7 or higher to ensure that this site would at the very least meet WCAG AA requirements.

The paragraphs of the site has been primarily kept in a left aligned fashion so as to not make the brain work harder since we learn to read left to right and are accustomed to text being laid out that way from an early age. Tables on the site have been properly captioned, images have been given alternative text, and all buttons, links, and forms have been coded to follow WAI-ARIA guidelines so those who rely on screen readers receive meaningful descriptions. The general font size I have chosen is 16 pixels. I chose this size because it is about the same size as text printed in a book or magazine, it is the size browsers display by default and were intended to display. As for the fonts themselves, I have chosen Verdana along with Arial, Tahoma, and San-serif as backup fonts. For my headers I have chosen Fredericka the Great because I really loved the look of it and that it was readable.

Structural Design:

Structurally the design is also simple, but accessible. I used level one headings at the top of each page and ensured that subsquent headings were done in order (i.e. level one, level two, level three, and so on). I used many semantic HTML5 elements such as article, aside, figcaption, figure, footer, header, main, nav, paragraph, section, and heading tags to structure the site. I used a skip link so that users who can only use a keyboard, screen readers, switch controls, and other assistive technologies can use the link to reach the main content of the site easier and faster.

As we live in a world of so many devices that come in a variety of different sizes, I designed the site to be responsive, meaning that the site will render well on all screen sizes and resolutions. I kept the structure consistent throughout the site to allow users to predict the location of certain elements because predictability is important for users with cognitive disabilities and helps create a more user-friendly experience for everyone. Lastly, my goal was to follow accessibility standards known as Web Content Accessibility Guidelines (WCAG). The structural design of this site will meet WCAG2.2 AA guidelines.
